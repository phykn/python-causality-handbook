# 6장. 그래프로 원인과 비교를 그려보기

## 말로만 하면 길을 놓친다

4장과 5장에서 우리는 비교가 망가지는 두 가지 방식을 봤다.

하나는 숨은 이유가 처치와 결과를 둘 다 움직이는 경우였다.

```text
게임에 진심인 정도 -> 비싼 마우스 사용
게임에 진심인 정도 -> 승률
```

다른 하나는 사람을 고르는 조건 때문에 이상한 관계가 생기는 경우였다.

```text
실력 -> 랭커 <- 장비
```

둘 다 말로 설명할 수는 있다.

하지만 변수가 조금만 많아지면 금방 헷갈린다.

비싼 마우스, 승률, 실력, 플레이 시간, 랭커 여부, 손목 피로, 팀원 수준이 한꺼번에 나오면 어디가 문제인지 놓치기 쉽다.

그래서 그림이 필요하다.

그래프는 멋있게 그리는 그림이 아니다.

> 어떤 길이 우리가 알고 싶은 길이고, 어떤 길이 평균 차이에 다른 이유를 섞는 길인지 표시하는 도구다.

## 화살표는 바꿀 수 있다는 뜻이다

그래프에서 화살표는 이렇게 읽는다.

```text
A -> B
```

이 말은 `A가 B를 바꿀 수 있다`는 뜻이다.

예를 들어 이런 화살표는 자연스럽다.

```text
비싼 마우스 사용 -> 승률
```

비싼 마우스를 쓰면 조작감이 좋아지고, 그 결과 승률이 바뀔 수 있다.

이것이 우리가 알고 싶은 길이다.

하지만 이런 화살표도 가능하다.

```text
게임에 진심인 정도 -> 비싼 마우스 사용
게임에 진심인 정도 -> 승률
```

게임에 진심인 사람은 장비를 살 가능성이 높다.

또 더 많이 연습해서 승률도 높을 수 있다.

그래프는 이 차이를 눈에 보이게 만든다.

## 알고 싶은 길과 다른 이유가 섞이는 길

마우스 효과를 알고 싶다면 가장 중요한 길은 이것이다.

```text
비싼 마우스 사용 -> 승률
```

이 차이는 보고 싶다.

마우스가 승률을 바꾸는 효과이기 때문이다.

그런데 이런 길은 문제다.

```text
비싼 마우스 사용 <- 게임에 진심인 정도 -> 승률
```

이 길은 마우스 효과가 아니다.

게임에 진심인 사람이 마우스도 사고 승률도 높기 때문에 생기는 길이다.

이 차이를 그대로 두면 평균 차이에 원래 실력 차이가 섞인다.

그래서 그래프를 볼 때는 먼저 이렇게 나눈다.

```text
알고 싶은 길: 처치 -> 결과
다른 이유가 섞이는 길: 처치 <- 다른 이유 -> 결과
```

그래프의 첫 역할은 길을 나누는 것이다.

## 맞춰야 할 차이가 보인다

다른 이유가 섞이는 길을 다시 보자.

```text
비싼 마우스 사용 <- 게임에 진심인 정도 -> 승률
```

이 문제를 줄이려면 `게임에 진심인 정도`가 비슷한 사람끼리 비교해야 한다.

진심인 사람끼리 비교한다.

가볍게 하는 사람끼리 비교한다.

그러면 이런 말이 가능해진다.

> 적어도 게임에 쏟는 관심은 비슷한 사람들 사이에서, 마우스가 다른가?

이것이 통제의 기본 생각이다.

통제는 변수를 많이 넣는 일이 아니다.

평균 차이에 다른 이유가 섞이지 않도록, 비교 전에 크게 달랐던 차이를 맞추는 일이다.

## 맞추면 안 되는 것도 있다

그런데 모든 차이를 맞추면 좋을까?

그렇지 않다.

예를 들어 이런 상황을 생각해 보자.

```text
비싼 마우스 사용 -> 손목 피로 감소 -> 승률
```

비싼 마우스가 손목 피로를 줄이고, 손목 피로가 줄어서 승률이 오를 수 있다.

이것은 마우스 효과가 지나가는 중간 결과일 수 있다.

그런데 손목 피로를 억지로 같다고 놓고 비교하면 어떻게 될까?

마우스가 승률을 바꾸는 과정 일부를 지워 버릴 수 있다.

그러면 효과를 너무 작게 볼 수 있다.

그래서 그래프는 이런 질문을 하게 만든다.

> 이 변수는 비교 전에 맞춰야 할 차이인가, 아니면 효과가 지나가는 중간 결과인가?

둘은 다르게 다뤄야 한다.

## 고르는 조건은 더 조심한다

5장에서 본 랭커 예시도 그래프로 다시 볼 수 있다.

```text
실력 -> 랭커 <- 장비
```

`랭커`는 두 이유가 만나는 지점이다.

실력이 좋아도 랭커가 될 수 있다.

장비가 좋아도 랭커가 될 수 있다.

이런 지점을 기준으로 사람을 고르면 원래 없던 관계가 생길 수 있다.

랭커만 보는 것은 이렇게 조건을 거는 일이다.

```text
랭커 = 예
```

그러면 장비가 부족한 랭커는 실력이 더 좋아야 하고, 실력이 부족한 랭커는 장비나 환경이 좋아야 한다.

이제 장비와 실력이 서로 이상하게 묶인다.

그래프는 여기서도 같은 일을 한다.

> 비교 대상을 고르는 조건이 관계를 새로 만들지 않는지 본다.

## 세 가지 모양만 먼저 구분하자

처음부터 모든 그래프 규칙을 외울 필요는 없다.

우선 세 가지 모양만 구분하면 된다.

| 모양 | 쉬운 말 | 조심할 점 |
| --- | --- | --- |
| `X <- Z -> Y` | Z가 X와 Y를 둘 다 움직인다 | Z를 맞춰야 비교가 쉬워진다 |
| `X -> Z -> Y` | X의 효과가 Z를 거쳐 Y로 간다 | Z를 맞추면 효과 일부를 지울 수 있다 |
| `X -> Z <- Y` | X와 Y가 Z에서 만난다 | Z로 고르면 없던 관계가 생길 수 있다 |

여기서 `X`는 처치다.

`Y`는 결과다.

`Z`는 중간에 있는 다른 변수다.

이 표의 핵심은 이름이 아니다.

같은 변수라도 위치에 따라 해야 할 일이 달라진다는 점이다.

## 직접 길을 적어 보면

아래 코드는 그래프를 계산하려는 것이 아니다.

화살표를 적어 놓고, 어떤 길이 알고 싶은 길이고 어떤 길이 다른 이유를 섞는 길인지 확인하는 작은 예시다.

```python
paths = {
    "알고 싶은 길": "비싼 마우스 사용 -> 승률",
    "맞춰야 할 차이": "비싼 마우스 사용 <- 게임에 진심인 정도 -> 승률",
    "맞추면 안 되는 중간 결과": "비싼 마우스 사용 -> 손목 피로 감소 -> 승률",
    "고르면 위험한 지점": "실력 -> 랭커 <- 장비",
}

for name, path in paths.items():
    print(name, ":", path)
```

코드는 단순하다.

중요한 것은 그래프를 그릴 때 길마다 역할을 붙이는 습관이다.

```text
알고 싶은 길: 효과를 보고 싶은 길
맞춰야 할 차이: 평균 차이에 다른 이유를 섞는 차이
맞추면 안 되는 중간 결과: 맞추면 효과 일부를 잃을 수 있는 결과
고르면 위험한 지점: 원래 없던 관계가 생길 수 있는 지점
```

그래프를 잘 그린다는 것은 예쁜 그림을 만드는 일이 아니다.

각 길이 비교에 어떤 일을 하는지 말할 수 있게 되는 것이다.

## 그래프가 답을 대신 주지는 않는다

그래프를 그렸다고 인과 효과가 자동으로 나오지는 않는다.

그래프는 자료가 아니라 우리의 생각을 그린 것이다.

그래서 틀린 생각으로 그리면 그래프도 틀린다.

예를 들어 `게임에 진심인 정도`가 승률에 영향을 준다고 생각했지만, 실제로는 별 상관이 없을 수도 있다.

반대로 중요한 변수를 빼먹을 수도 있다.

그래도 그래프는 유용하다.

왜냐하면 우리가 어떤 비교를 믿고 있는지 드러내기 때문이다.

그래프를 그리면 이런 질문을 피할 수 없다.

> 이 화살표는 정말 말이 되는가?
>
> 이 차이는 비교 전에 맞춰야 하는가?
>
> 이 조건을 걸면 비교가 바뀌는가?

이 질문들이 바로 인과추론의 핵심이다.

## 여기서 다음 질문이 생긴다

그래프를 보면 어떤 차이를 맞춰야 할지 보인다.

하지만 실제 자료에서는 차이를 손으로 맞출 수 없다.

우리가 할 수 있는 일은 비슷한 사람끼리 비교하거나, 계산으로 조건을 맞추는 것이다.

다음 장에서는 그 대표적인 도구를 본다.

> 회귀는 어떻게 비슷한 사람끼리 비교하게 도와줄까?

## 한 줄 요약

인과 그래프는 원인 관계를 예쁘게 그리는 그림이 아니라, 알고 싶은 길과 평균 차이에 다른 이유를 섞는 길을 구분해서 어떤 비교를 믿을 수 있는지 점검하는 도구다.
