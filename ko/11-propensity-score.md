# 11장. 처치받을 가능성으로 균형 맞추기

## 조건을 하나씩 맞추기 어렵다면

10장에서는 비슷한 사람끼리 짝을 붙였다.

비싼 마우스를 산 사람에게, 원래 승률과 플레이 시간이 비슷한 기존 마우스 사용자를 붙였다.

이 생각은 직관적이다.

하지만 조건이 많아지면 짝을 찾기 어려워진다.

```text
원래 승률
플레이 시간
게임 모드
포지션
팀원 수준
접속 시간대
장비에 쓰는 돈
```

이 조건을 하나씩 모두 맞추려면 자료가 금방 부족해진다.

그래서 다른 질문을 해 볼 수 있다.

> 이 사람이 비싼 마우스를 살 가능성은 얼마나 높았을까?

조건을 하나씩 직접 맞추는 대신, 여러 조건을 하나의 가능성 점수로 줄여 보는 것이다.

## 살 것 같은 사람과 안 살 것 같은 사람

장비 회사가 구매 기록을 보니 이런 패턴이 보인다고 하자.

```text
원래 승률이 높을수록
플레이 시간이 많을수록
게임에 진심일수록
비싼 마우스를 살 가능성이 높다.
```

그러면 어떤 사람은 처음부터 비싼 마우스를 살 것 같았다.

어떤 사람은 살 가능성이 낮았다.

예를 들어 이렇게 볼 수 있다.

| 플레이어 | 원래 승률 | 플레이 시간 | 진심인 정도 | 비싼 마우스를 살 가능성 |
| --- | ---: | ---: | --- | ---: |
| A | 70% | 4.0시간 | 높음 | 0.85 |
| B | 68% | 3.8시간 | 높음 | 0.82 |
| C | 55% | 2.0시간 | 보통 | 0.45 |
| D | 53% | 2.1시간 | 보통 | 0.47 |
| E | 42% | 0.8시간 | 낮음 | 0.12 |
| F | 40% | 0.7시간 | 낮음 | 0.10 |

여기서 0.85는 이렇게 읽으면 된다.

```text
이 조건을 가진 사람은 비싼 마우스를 살 가능성이 약 85%다.
```

중요한 점은 이 점수가 승률을 직접 예측하는 점수가 아니라는 것이다.

이 점수는 처치를 받을 가능성, 여기서는 비싼 마우스를 살 가능성을 말한다.

## 같은 가능성끼리 비교한다

이제 A와 F를 비교하면 어떨까?

A는 비싼 마우스를 살 가능성이 0.85다.

F는 0.10이다.

두 사람은 처음부터 너무 다르다.

비싼 마우스를 썼는지 말고도, 게임에 진심인 정도와 플레이 시간이 크게 다를 가능성이 높다.

반대로 C와 D를 보자.

C의 가능성은 0.45다.

D의 가능성은 0.47이다.

두 사람은 비싼 마우스를 살 가능성이 비슷하다.

그런데 C는 실제로 비싼 마우스를 샀고, D는 사지 않았다고 하자.

그러면 이렇게 물을 수 있다.

> 둘 다 비싼 마우스를 살 가능성은 비슷했는데, 실제 선택만 달랐다면 비교가 조금 더 공정하지 않을까?

이것이 성향 점수의 핵심 직관이다.

조건을 하나하나 모두 맞추기 어렵다면, 처치를 받을 가능성이 비슷한 사람끼리 비교한다.

## 그 가능성을 성향 점수라고 부른다

처치를 받을 가능성을 **성향 점수**라고 부른다.

영어로는 `propensity score`다.

기호로는 보통 이렇게 쓴다.

```text
e(x) = P(T = 1 | X = x)
```

처음 보면 복잡해 보이지만 말은 단순하다.

`T = 1`은 처치를 받았다는 뜻이다.

이 장에서는 비싼 마우스를 샀다는 뜻이다.

`X = x`는 그 사람의 조건을 본다는 뜻이다.

원래 승률, 플레이 시간, 진심인 정도 같은 조건이다.

그래서 이 식은 이렇게 읽으면 된다.

```text
그런 조건을 가진 사람이 비싼 마우스를 살 가능성
```

성향 점수는 결과가 아니라 선택을 설명한다.

승률이 얼마나 될지를 맞히는 점수가 아니다.

누가 비싼 마우스를 살 것 같았는지를 나타내는 점수다.

## 왜 이 점수가 도움이 될까

성향 점수가 같은 두 사람을 생각해 보자.

둘 다 비싼 마우스를 살 가능성이 0.50이다.

그런데 한 사람은 샀고, 한 사람은 사지 않았다.

둘의 조건이 완전히 같다고 말할 수는 없다.

하지만 “비싼 마우스를 살 것 같은 정도”는 비슷하다.

이렇게 보면 비교가 조금 더 공정해진다.

```text
비싼 마우스를 살 가능성이 비슷한 사람끼리
실제로 산 사람과 사지 않은 사람을 비교한다.
```

이 생각은 10장의 매칭과 이어진다.

매칭은 사람을 직접 찾아 붙인다.

성향 점수는 먼저 사람의 조건을 하나의 가능성 점수로 줄인다.

그 다음 비슷한 점수끼리 비교한다.

## 드문 사람이 더 크게 반영된다

성향 점수는 짝짓기에만 쓰이지 않는다.

사람들에게 가중치를 주는 데도 쓴다.

가중치는 어떤 사람의 자료를 얼마나 크게 볼지 정하는 숫자다.

예를 들어 이런 사람이 있다고 하자.

```text
비싼 마우스를 살 가능성은 0.10인데
실제로는 비싼 마우스를 샀다.
```

이 사람은 드문 경우다.

보통은 비싼 마우스를 안 살 것 같은 사람인데 샀기 때문이다.

그래서 이 사람은 중요하다.

기존 마우스 사용자와 비슷하게 생긴 사람 중에서, 실제로 비싼 마우스를 쓴 사례이기 때문이다.

반대로 이런 사람도 중요하다.

```text
비싼 마우스를 살 가능성은 0.90인데
실제로는 사지 않았다.
```

이 사람도 드문 경우다.

비싼 마우스를 살 것 같은 사람인데 사지 않았기 때문이다.

그래서 비싼 마우스 사용자와 비슷한 비교 대상이 된다.

성향 점수 가중치는 이런 드문 사람을 더 크게 본다.

## 가중치는 이렇게 읽는다

성향 점수가 `e`라고 하자.

비싼 마우스를 산 사람의 가중치는 이렇게 둔다.

```text
1 / e
```

비싼 마우스를 사지 않은 사람의 가중치는 이렇게 둔다.

```text
1 / (1 - e)
```

왜 이렇게 할까?

가능성이 낮은데 처치를 받은 사람은 드물다.

예를 들어 `e = 0.10`인데 비싼 마우스를 샀다면 가중치는 10이다.

```text
1 / 0.10 = 10
```

가능성이 높은데 처치를 받지 않은 사람도 드물다.

예를 들어 `e = 0.90`인데 기존 마우스를 썼다면 가중치는 10이다.

```text
1 / (1 - 0.90) = 10
```

이 방법을 **처치 확률의 역수 가중치**라고 부른다.

영어로는 `inverse probability of treatment weighting`이다.

보통 줄여서 `IPTW`라고 쓴다.

이름은 길지만 핵심은 짧다.

> 원래 그 선택을 할 것 같지 않았던 사람을 더 크게 본다.

## 작은 표로 계산해 보면

아래 표에는 비싼 마우스를 산 사람과 사지 않은 사람이 섞여 있다.

| 플레이어 | 비싼 마우스 사용 | 성향 점수 | 승률 |
| --- | ---: | ---: | ---: |
| A | 1 | 0.80 | 72% |
| B | 1 | 0.60 | 66% |
| C | 1 | 0.20 | 58% |
| D | 0 | 0.75 | 67% |
| E | 0 | 0.40 | 55% |
| F | 0 | 0.15 | 47% |

그냥 평균을 비교하면 이렇게 된다.

```text
비싼 마우스 사용자 평균 = 65.3%
기존 마우스 사용자 평균 = 56.3%
차이 = 9.0%p
```

하지만 성향 점수 가중치를 쓰면 드문 사람이 더 크게 반영된다.

C는 성향 점수가 0.20인데 실제로 비싼 마우스를 썼다.

그래서 비싼 마우스를 잘 안 살 것 같은 사람 중에서 중요한 사례다.

D는 성향 점수가 0.75인데 실제로 기존 마우스를 썼다.

그래서 비싼 마우스를 살 것 같은 사람 중에서 중요한 비교 대상이다.

이렇게 다시 평균을 내면 비교가 달라질 수 있다.

## 가중 평균을 내 보면

아래 코드는 방금 표에서 가중 평균을 계산한다.

새로운 이론을 코드로 배우는 것이 아니라, 위에서 말한 가중치가 어떻게 평균을 바꾸는지 확인하는 코드다.

```python
players = [
    {"name": "A", "treated": 1, "score": 0.80, "win": 72},
    {"name": "B", "treated": 1, "score": 0.60, "win": 66},
    {"name": "C", "treated": 1, "score": 0.20, "win": 58},
    {"name": "D", "treated": 0, "score": 0.75, "win": 67},
    {"name": "E", "treated": 0, "score": 0.40, "win": 55},
    {"name": "F", "treated": 0, "score": 0.15, "win": 47},
]


def average(rows):
    return sum(row["win"] for row in rows) / len(rows)


def weighted_average(rows):
    total_weight = sum(row["weight"] for row in rows)
    return sum(row["win"] * row["weight"] for row in rows) / total_weight


for row in players:
    if row["treated"]:
        row["weight"] = 1 / row["score"]
    else:
        row["weight"] = 1 / (1 - row["score"])

treated = [row for row in players if row["treated"]]
untreated = [row for row in players if not row["treated"]]

plain_gap = average(treated) - average(untreated)
weighted_gap = weighted_average(treated) - weighted_average(untreated)

round(plain_gap, 2), round(weighted_gap, 2)
```

결과는 이렇게 읽으면 된다.

```text
그냥 평균 차이 = 9.0
가중 평균 차이 = 1.26
```

숫자 하나가 중요한 것이 아니다.

중요한 것은 평균을 내는 방식이 바뀌었다는 점이다.

성향 점수 가중치는 “비교에 더 필요한 사람”을 더 크게 보려고 한다.

## 점수는 직접 추정해야 한다

여기서 조심해야 할 점이 있다.

성향 점수는 보통 실제로 알고 있는 값이 아니다.

우리가 자료로 추정한다.

예를 들어 이런 정보를 넣어 “비싼 마우스를 살 가능성”을 맞혀 본다.

```text
원래 승률
플레이 시간
진심인 정도
장비 구매 이력
게임 모드
```

이때 쓰는 모델은 로지스틱 회귀일 수도 있고, 다른 예측 모델일 수도 있다.

하지만 모델이 틀리면 성향 점수도 틀린다.

성향 점수가 틀리면 가중치도 틀린다.

그러면 결과도 흔들린다.

그래서 성향 점수는 “계산하면 끝나는 정답”이 아니다.

어떤 조건으로 처치 가능성을 추정했는지 설명할 수 있어야 한다.

## 너무 큰 가중치는 위험하다

성향 점수 가중치에는 또 하나의 문제가 있다.

점수가 0이나 1에 너무 가까우면 가중치가 커진다.

예를 들어 비싼 마우스를 살 가능성이 0.01인 사람이 실제로 비싼 마우스를 샀다고 하자.

가중치는 100이다.

```text
1 / 0.01 = 100
```

한 사람이 평균을 너무 크게 흔들 수 있다.

반대로 비싼 마우스를 살 가능성이 0.99인 사람이 실제로 사지 않아도 가중치는 100이다.

이런 자료는 분석을 불안정하게 만든다.

그래서 성향 점수를 볼 때는 극단적인 점수가 많은지 확인해야 한다.

## 비교할 양쪽이 모두 있어야 한다

10장에서 본 문제도 그대로 남는다.

비슷한 조건 안에 양쪽 사람이 모두 있어야 한다.

비싼 마우스를 살 가능성이 0.95인 사람 중에 전부 비싼 마우스를 샀다면 어떨까?

그 구간에는 기존 마우스 사용자가 없다.

그러면 비교할 대상이 없다.

반대로 가능성이 0.05인 사람 중에 전부 기존 마우스를 썼다면, 그 구간에는 비싼 마우스 사용자가 없다.

이런 경우 성향 점수를 써도 비교가 어렵다.

이 조건을 **positivity**라고 부른다.

한국어로는 “양쪽 가능성이 모두 있어야 한다” 정도로 이해하면 된다.

> 같은 조건의 사람들 안에서 처치를 받은 사람도 있고, 받지 않은 사람도 있어야 한다.

## 점수는 보이지 않는 차이를 못 본다

성향 점수도 관측한 조건으로 만든다.

자료에 없는 차이는 점수에 들어가지 않는다.

예를 들어 장비 세팅을 얼마나 잘하는지 자료에 없다고 하자.

그런데 이 능력이 비싼 마우스 구매와 승률을 둘 다 움직인다면 문제가 남는다.

성향 점수는 그 차이를 맞추지 못한다.

그래서 성향 점수는 숨은 이유를 자동으로 없애는 도구가 아니다.

관측한 조건을 이용해 비교를 더 공정하게 만들려는 방법이다.

## 매칭과 같은 질문을 다른 방식으로 묻는다

매칭은 비슷한 사람을 직접 찾는다.

성향 점수는 비슷한 가능성을 가진 사람을 찾거나, 그 가능성으로 가중치를 준다.

모양은 다르지만 질문은 같다.

```text
처치를 받은 사람과 받지 않은 사람이 비교 가능하도록 만들 수 있는가?
```

성향 점수는 이 질문에 이렇게 답한다.

```text
처치를 받을 가능성이 비슷한 사람끼리 비교해 보자.
또는 드문 선택을 한 사람에게 더 큰 가중치를 주자.
```

그래서 성향 점수는 회귀나 매칭을 대체하는 마법이 아니다.

관측 자료에서 비교를 만드는 또 하나의 방법이다.

## 하나의 모델만 믿어도 될까

여기까지 오면 새로운 걱정이 생긴다.

성향 점수는 처치를 받을 가능성 모델에 크게 기대고 있다.

그 모델이 틀리면 결과도 흔들릴 수 있다.

그렇다면 다른 안전장치를 붙일 수 있을까?

예를 들어 결과를 예측하는 모델도 같이 쓰면 어떨까?

처치 가능성 모델과 결과 모델 중 하나라도 괜찮으면 추정이 크게 무너지지 않게 만들 수 있을까?

다음 장에서는 이 생각을 본다.

## 한 줄 요약

성향 점수는 여러 조건을 “처치를 받을 가능성” 하나로 줄여 비교를 맞추려는 방법이며, 점수가 틀리거나 비교할 양쪽 사람이 없거나 보이지 않는 차이가 있으면 여전히 위험하다.
