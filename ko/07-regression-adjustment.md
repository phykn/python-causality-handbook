# 7장. 회귀를 조건부 비교로 읽기

## 사람을 손으로 나누기엔 너무 많다

6장에서 우리는 그래프로 이런 질문을 했다.

> 어떤 차이를 맞춰야 하고, 어떤 차이는 건드리면 안 될까?

장비 예시에서는 `게임에 진심인 정도`가 중요했다.

진심인 사람은 비싼 마우스를 살 가능성이 높다.

그리고 승률도 높을 가능성이 크다.

그래서 비싼 마우스 사용자 전체와 기존 마우스 사용자 전체를 바로 비교하면 위험했다.

더 나은 비교는 이것이었다.

```text
진심인 사람끼리 비교한다.
가볍게 하는 사람끼리 비교한다.
```

문제는 현실의 자료가 이렇게 깔끔하지 않다는 점이다.

플레이 시간, 원래 승률, 포지션, 게임 모드, 팀원 수준까지 같이 맞춰야 한다면 어떻게 할까?

사람을 손으로 하나씩 나누다 보면 금방 복잡해진다.

회귀는 여기서 필요해진다.

회귀를 먼저 이렇게 읽자.

> 비슷한 조건끼리 비교한 차이를 계산해서 한 숫자로 요약하는 방법.

## 전체 평균은 아직 위험하다

다시 작은 자료를 보자.

| 플레이어 유형 | 마우스 | 평균 승률 |
| --- | --- | ---: |
| 진심인 사람 | 비싼 마우스 | 70% |
| 진심인 사람 | 기존 마우스 | 65% |
| 가볍게 하는 사람 | 비싼 마우스 | 50% |
| 가볍게 하는 사람 | 기존 마우스 | 45% |

각 유형 안에서 마우스 차이는 5%p다.

```text
진심인 사람 안에서: 70% - 65% = 5%p
가볍게 하는 사람 안에서: 50% - 45% = 5%p
```

그런데 실제 자료는 이렇게 섞여 있을 수 있다.

| 그룹 | 진심인 사람 | 가볍게 하는 사람 |
| --- | ---: | ---: |
| 비싼 마우스 그룹 | 8명 | 2명 |
| 기존 마우스 그룹 | 2명 | 8명 |

비싼 마우스 그룹에는 진심인 사람이 많다.

기존 마우스 그룹에는 가볍게 하는 사람이 많다.

이 상태에서 전체 평균만 비교하면 차이가 커진다.

```text
비싼 마우스 그룹 평균 = 66%
기존 마우스 그룹 평균 = 49%
전체 평균 차이 = 17%p
```

하지만 우리가 앞에서 확인한 마우스 효과는 유형 안에서 5%p였다.

17%p는 마우스 효과만 말하지 않는다.

그 안에는 `게임에 진심인 정도` 차이가 섞여 있다.

## 회귀는 같은 조건 안의 차이를 본다

회귀가 하는 일을 말로 풀면 이렇다.

> 게임에 진심인 정도가 비슷한 사람들 사이에서, 마우스가 다르면 승률이 얼마나 다른가?

이 질문은 전체 평균 차이와 다르다.

전체 평균 차이는 이렇게 묻는다.

```text
비싼 마우스 사용자 전체와 기존 마우스 사용자 전체가 얼마나 다른가?
```

회귀로 맞춘 비교는 이렇게 묻는다.

```text
진심인 정도를 같다고 보면, 마우스 차이는 얼마나 남는가?
```

여기서 `진심인 정도를 같다고 본다`는 말은 실제로 모든 사람을 똑같이 만든다는 뜻이 아니다.

계산할 때 그 차이를 따로 빼고 보겠다는 뜻이다.

그래서 회귀는 통제의 계산 버전처럼 읽을 수 있다.

## 계수는 조건을 맞춘 뒤 남은 차이다

회귀식은 처음 보면 딱딱하다.

하지만 뜻은 단순하다.

```text
승률 = 기준 승률
     + 마우스 계수 × 비싼 마우스 사용
     + 진심 계수 × 진심인 사람
     + 남은 차이
```

여기서 `비싼 마우스 사용`은 비싼 마우스를 쓰면 1, 아니면 0이라고 표시한다.

`진심인 사람`도 진심인 사람이면 1, 아니면 0이라고 표시한다.

이때 **마우스 계수**는 이렇게 읽는다.

> 진심인 정도를 비교에서 맞춘 뒤에도, 비싼 마우스 사용자가 평균적으로 얼마나 더 높은가?

계수는 그냥 모델이 뱉은 숫자가 아니다.

우리가 어떤 조건을 맞춘 뒤 남은 차이를 보겠다고 정했을 때 나오는 요약값이다.

그래서 회귀 계수를 볼 때는 먼저 물어야 한다.

> 무엇을 맞춘 뒤의 차이인가?

## 숫자로 보면 차이가 줄어든다

앞의 표에서는 전체 평균 차이가 17%p였다.

하지만 유형 안에서 비교하면 둘 다 5%p였다.

회귀가 제대로 맞춰 준다면 마우스 계수는 17%p가 아니라 5%p에 가까워져야 한다.

작은 자료에서는 손으로도 확인할 수 있다.

```text
진심인 사람 안의 마우스 차이 = 5%p
가볍게 하는 사람 안의 마우스 차이 = 5%p

조건을 맞춘 뒤 남은 차이 = 5%p
```

이것이 회귀를 인과추론에서 쓰는 핵심 이유다.

회귀는 단순히 예측선을 만드는 도구가 아니다.

비슷한 조건끼리 비교한 차이를 한 숫자로 요약하는 도구다.

## 직접 맞춰 보면

아래 코드는 회귀가 하려는 일을 손으로 흉내 낸다.

먼저 전체 평균 차이를 계산한다.

그다음 진심인 사람 안에서, 가볍게 하는 사람 안에서 따로 비교한다.

```python
players = []

def add_players(player_type, mouse, win_rate, count):
    for _ in range(count):
        players.append({
            "type": player_type,
            "mouse": mouse,
            "win_rate": win_rate,
        })

add_players("serious", "expensive", 70, 8)
add_players("casual", "expensive", 50, 2)
add_players("serious", "basic", 65, 2)
add_players("casual", "basic", 45, 8)

def average(rows):
    return sum(row["win_rate"] for row in rows) / len(rows)

expensive = [row for row in players if row["mouse"] == "expensive"]
basic = [row for row in players if row["mouse"] == "basic"]
overall_gap = average(expensive) - average(basic)

gaps = []
for player_type in ["serious", "casual"]:
    type_expensive = [
        row for row in players
        if row["type"] == player_type and row["mouse"] == "expensive"
    ]
    type_basic = [
        row for row in players
        if row["type"] == player_type and row["mouse"] == "basic"
    ]
    gaps.append(average(type_expensive) - average(type_basic))

adjusted_gap = sum(gaps) / len(gaps)

overall_gap, adjusted_gap
```

결과는 이렇게 읽으면 된다.

```text
전체 평균 차이 = 17%p
조건을 맞춘 뒤 차이 = 5%p
```

이 코드는 진짜 회귀 함수를 쓰지는 않는다.

하지만 회귀가 왜 필요한지는 보여준다.

전체로 한 번에 비교하면 17%p처럼 보인다.

비슷한 조건 안에서 비교하면 5%p가 남는다.

회귀 계수는 이런 조건부 비교를 더 많은 변수에 대해 계산해 주는 숫자다.

## 맞춘다고 다 해결되지는 않는다

회귀를 쓰면 숫자가 그럴듯해진다.

그래서 더 조심해야 한다.

회귀는 우리가 넣은 조건만 맞춘다.

중요한 조건을 빼먹으면 여전히 비교가 틀릴 수 있다.

예를 들어 `게임에 진심인 정도`를 빼고 마우스만 넣으면, 회귀도 17%p에 가까운 차이를 효과처럼 보여줄 수 있다.

반대로 맞추면 안 되는 변수를 넣어도 문제가 된다.

마우스를 바꾼 뒤 생긴 손목 피로를 억지로 맞추면, 마우스 효과의 일부를 지워 버릴 수 있다.

그래서 회귀를 돌리기 전에 그래프 질문이 먼저다.

```text
무엇을 맞춰야 하는가?
무엇을 맞추면 안 되는가?
무엇을 빼먹으면 비교가 틀어지는가?
```

회귀는 이 질문에 답해 주지 않는다.

답을 정한 뒤 계산을 도와줄 뿐이다.

## 여기서 다음 질문이 생긴다

이번 장에서는 회귀를 조건부 비교로 읽었다.

하지만 아직 한 가지가 남아 있다.

`진심인 사람`, `가볍게 하는 사람`처럼 말로 된 집단은 회귀식에 어떻게 넣을까?

다음 장에서는 집단 이름을 0과 1로 바꿔서 회귀가 집단 비교를 어떻게 처리하는지 본다.

> 더미 변수는 어떻게 집단 비교를 계산으로 바꿀까?

## 한 줄 요약

회귀는 인과 효과를 자동으로 찾아주는 도구가 아니라, 우리가 맞추기로 한 조건 안에서 처치와 결과의 차이를 비교해 한 숫자로 요약하는 도구다.
