# 8장. 더미 변수와 집단 비교

## 계산은 이름표를 그대로 읽지 못한다

7장에서 우리는 이런 질문을 했다.

> 조건이 비슷하다고 보면, 마우스 차이는 얼마나 남을까?

여기서 조건 중 하나는 `게임 참여도`였다. 예를 들어 사람을 두 집단으로 나눌 수 있다.

```text
게임 참여도 높은 사람
게임 참여도 낮은 사람
```

사람은 이 이름표를 바로 이해한다. 하지만 회귀는 글자 이름표를 그대로 계산하지 못한다. 회귀는 숫자로 계산한다.

그래서 집단 이름을 숫자 표시로 바꿔야 한다. 이때 쓰는 것이 **더미 변수**다. 영어로는 `dummy variable`이다.

더미 변수의 역할은 한 문장으로 말할 수 있다.

> 어떤 조건에 해당하면 1, 해당하지 않으면 0으로 표시한다.

## 해당하면 1, 아니면 0

예를 들어 `게임 참여도 높은 사람`이라는 조건을 숫자로 바꿔 보자.

| 플레이어 | 유형 | 참여도 높음 표시 |
| --- | --- | ---: |
| A | 게임 참여도 높음 | 1 |
| B | 게임 참여도 높음 | 1 |
| C | 게임 참여도 낮음 | 0 |
| D | 게임 참여도 낮음 | 0 |

`참여도 높음 표시`가 더미 변수다. 게임 참여도가 높으면 1이고, 낮으면 0이다.

마우스도 같은 방식으로 표시할 수 있다.

| 플레이어 | 마우스 | 비싼 마우스 표시 |
| --- | --- | ---: |
| A | 비싼 마우스 | 1 |
| B | 기존 마우스 | 0 |

이제 회귀는 글자가 아니라 숫자를 볼 수 있다.

```text
게임 참여도 높음 -> 1
게임 참여도 낮음 -> 0

비싼 마우스 -> 1
기존 마우스 -> 0
```

더미 변수는 어려운 통계 장치가 아니라, 집단 이름을 계산 가능한 표시로 바꾸는 방법이다.

## 0은 버린다는 뜻이 아니라 기준이라는 뜻이다

여기서 가장 먼저 확인할 지점이 있다. `게임 참여도 낮음 = 0`이라고 해서, 그 집단이 중요하지 않다는 뜻은 아니다.

0은 기준이라는 뜻이다.

예를 들어 이렇게 정했다.

```text
게임 참여도 높음 = 1
게임 참여도 낮음 = 0
```

그러면 `게임 참여도 낮음`이 기준 집단이 된다. 회귀는 이렇게 묻는다.

> 기준인 게임 참여도 낮은 사람과 비교해서, 게임 참여도 높은 사람은 승률이 얼마나 다른가?

만약 `참여도 높음 표시`의 계수가 20이라면 이렇게 읽는다.

```text
게임 참여도 낮은 사람보다
게임 참여도 높은 사람의 승률이 평균 20%p 높다.
```

0은 버린 집단이 아니다. 비교의 출발점이다.

## 기준에서 하나씩 더한다

더미 변수를 넣은 회귀는 더하기표로 읽으면 된다.

아주 단순한 예를 보자.

```text
기준 승률 = 45%
게임 참여도 높음 차이 = +20%p
비싼 마우스 차이 = +5%p
```

기준은 `게임 참여도 낮음 + 기존 마우스`라고 하자. 그러면 각 경우는 이렇게 계산된다.

| 유형 | 마우스 | 계산 | 예상 승률 |
| --- | --- | --- | ---: |
| 게임 참여도 낮음 | 기존 마우스 | 45 | 45% |
| 게임 참여도 낮음 | 비싼 마우스 | 45 + 5 | 50% |
| 게임 참여도 높음 | 기존 마우스 | 45 + 20 | 65% |
| 게임 참여도 높음 | 비싼 마우스 | 45 + 20 + 5 | 70% |

이 표는 7장에서 본 숫자와 같다. 게임 참여도가 높은 사람 안에서도 마우스 차이는 5%p다. 게임 참여도가 낮은 사람 안에서도 마우스 차이는 5%p다.

더미 변수는 이 비교를 계산식 안에 넣는 방법이다.

## 회귀식도 더하기표로 읽는다

회귀식으로 쓰면 이렇게 보인다.

```text
승률 = 45
     + 20 × 참여도 높음 표시
     + 5 × 비싼 마우스 표시
```

처음 보면 수식 같지만, 방금 본 더하기표를 짧게 쓴 것이다.

게임 참여도가 낮은 사람이 기존 마우스를 쓰면 두 표시가 모두 0이다.

```text
승률 = 45 + 20 × 0 + 5 × 0
     = 45
```

게임 참여도가 높은 사람이 비싼 마우스를 쓰면 두 표시가 모두 1이다.

```text
승률 = 45 + 20 × 1 + 5 × 1
     = 70
```

계산기는 이렇게 0과 1을 넣어 집단 차이를 더한다. 그래서 더미 변수는 집단 이름을 회귀식 안에 넣는 방법이다.

## 두 집단이면 하나만 표시해도 된다

집단이 두 개라면 더미 변수 하나면 충분하다.

```text
참여도 높음 표시 = 1이면 게임 참여도 높음
참여도 높음 표시 = 0이면 게임 참여도 낮음
```

그런데 이런 식으로 둘 다 넣으려고 하면 문제가 생긴다.

```text
참여도 높음 표시
참여도 낮음 표시
```

둘 중 하나는 항상 다른 하나의 반대다. 참여도가 높으면 참여도가 낮지 않고, 참여도가 낮으면 참여도가 높지 않다.

그래서 두 집단 중 하나는 기준으로 두고, 나머지 하나만 표시해도 된다.

여기서는 이 정도만 기억하면 충분하다.

> 집단이 두 개면, 하나를 기준으로 두고 다른 하나를 1로 표시한다.

집단이 더 많아도 생각은 같다. 셋 중 하나를 기준으로 두고, 나머지 집단을 각각 표시한다.

## 숫자로 바꾸면 앞의 표를 계산으로 만들 수 있다

이름표를 0과 1로 바꾸면 앞의 표와 같은 구조를 더하기로 만들 수 있다.

```text
participation_high expensive 70
participation_high basic 65
participation_low expensive 50
participation_low basic 45
```

여기서 중요한 것은 영어 이름이 아니다. 더미 변수가 집단 이름을 0과 1로 바꾸고, 회귀식이 그 표시를 더해서 집단 평균을 만든다는 점이다.

## 더미 변수가 답을 보장하지는 않는다

더미 변수는 계산을 가능하게 해 준다. 하지만 어떤 집단을 표시해야 하는지는 여전히 우리가 판단해야 한다.

`게임 참여도 높음`을 표시하면 게임 참여도 차이를 맞추는 데 도움이 될 수 있다. 비교 전에 이미 달랐고, 마우스 사용과 승률을 둘 다 바꿀 수 있기 때문이다.

하지만 맞추면 안 되는 중간 결과를 더미 변수로 넣으면 문제가 생길 수 있다. 예를 들어 `마우스를 바꾼 뒤 손목이 덜 아픈 사람`을 표시해서 맞춰 버리면, 마우스 효과의 일부를 지울 수 있다.

그래서 더미 변수도 회귀와 마찬가지다. 계산은 도와주지만, 무엇을 넣을지는 그래프 질문이 먼저다.

## 무엇을 넣고 빼야 할까

이제 우리는 집단 이름을 회귀식에 넣는 방법을 봤다. 회귀는 조건을 맞추는 데 도움이 되고, 더미 변수는 집단 이름을 숫자로 바꿔 회귀가 다룰 수 있게 해 준다.

하지만 아직 중요한 질문이 남아 있다.

> 어떤 변수는 넣어야 하고, 어떤 변수는 넣으면 안 될까?

다음 장에서는 좋은 통제와 잘못된 통제를 더 분명하게 나눈다.

## 한 줄 요약

더미 변수는 집단 이름을 0과 1의 표시로 바꿔서, 회귀가 기준 집단과 다른 집단의 차이를 계산할 수 있게 해 주는 방법이다.
