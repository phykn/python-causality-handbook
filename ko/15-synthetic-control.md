# 15장. 비교할 PC방이 하나도 딱 맞지 않다면

## 하나를 고르려고 하면 계속 걸린다

장비 회사가 이번에는 한 PC방에만 새 마우스를 먼저 설치했다.

이 PC방을 `스타 PC방`이라고 하자.

회사 회의실에 질문이 올라온다.

```text
스타 PC방의 승률이 오른 것이 새 마우스 때문인가?
```

자료는 여러 주 동안 있다.

새 마우스는 5주차부터 들어갔다.

| PC방 | 1주차 | 2주차 | 3주차 | 4주차 | 5주차 | 6주차 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 스타 PC방 | 52% | 54% | 55% | 57% | 66% | 68% |
| B PC방 | 48% | 49% | 50% | 51% | 52% | 53% |
| C PC방 | 56% | 57% | 59% | 60% | 61% | 62% |
| D PC방 | 51% | 53% | 54% | 56% | 57% | 58% |

쉬운 방법은 비교 PC방 하나를 고르는 것이다.

하지만 바로 문제가 생긴다.

B PC방은 스타 PC방보다 계속 낮다.

C PC방은 스타 PC방보다 계속 높다.

D PC방은 꽤 비슷하지만, 그래도 완전히 같지는 않다.

하나만 고르면 부족한 점이 남는다.

```text
B는 너무 낮다.
C는 너무 높다.
D는 비슷하지만 흐름이 조금 다르다.
```

그렇다면 질문을 바꿔 볼 수 있다.

```text
비교 PC방 하나를 고르지 말고,
여러 PC방을 조금씩 섞으면 스타 PC방과 비슷해질까?
```

## 여러 곳을 섞어 하나처럼 만든다

예를 들어 이렇게 섞어 보자.

```text
B PC방 20%
C PC방 30%
D PC방 50%
```

그러면 1주차 비교 대상은 이렇게 계산된다.

```text
B 48% * 0.2 + C 56% * 0.3 + D 51% * 0.5
= 9.6% + 16.8% + 25.5%
= 51.9%
```

스타 PC방의 1주차 승률은 52%였다.

51.9%면 꽤 가깝다.

같은 방식으로 1주차부터 4주차까지 계산하면 이렇다.

| 주차 | 스타 PC방 | 섞어서 만든 비교 대상 |
| --- | ---: | ---: |
| 1주차 | 52% | 51.9% |
| 2주차 | 54% | 53.6% |
| 3주차 | 55% | 55.3% |
| 4주차 | 57% | 57.2% |

새 마우스가 들어가기 전에는 두 흐름이 꽤 비슷하다.

이제 이 섞은 대상을 이렇게 읽을 수 있다.

```text
새 마우스를 설치하지 않았다면
스타 PC방은 이 비교 대상처럼 움직였을지도 모른다.
```

이런 식으로 비율을 붙여 평균을 내는 것을 가중 평균이라고 부른다.

여기서 만든 가중 평균 비교 대상이 **합성 통제집단**이다.

영어로는 `synthetic control`이다.

말 그대로 여러 통제 대상을 섞어서 만든 비교 대상이다.

## 설치 전을 먼저 맞춘다

합성 통제집단에서 가장 중요한 확인은 설치 전이다.

설치 후를 맞추면 안 된다.

우리가 알고 싶은 것이 설치 후 차이이기 때문이다.

그래서 순서는 이렇다.

```text
1. 새 마우스 설치 전 자료만 본다.
2. 스타 PC방과 비슷한 흐름이 되도록 다른 PC방의 비율을 정한다.
3. 그 비율을 설치 후에도 그대로 사용한다.
4. 설치 후 스타 PC방과 합성 통제집단의 차이를 본다.
```

위 예시에서는 설치 전 4주 동안 이런 비율을 골랐다.

| PC방 | 비율 |
| --- | ---: |
| B PC방 | 20% |
| C PC방 | 30% |
| D PC방 | 50% |

그 비율을 설치 후에도 그대로 쓴다.

5주차 합성 통제집단은 이렇게 된다.

```text
B 52% * 0.2 + C 61% * 0.3 + D 57% * 0.5
= 10.4% + 18.3% + 28.5%
= 57.2%
```

그런데 스타 PC방의 5주차 승률은 66%다.

차이는 이렇다.

```text
66% - 57.2% = 8.8%p
```

6주차도 계산해 보자.

```text
B 53% * 0.2 + C 62% * 0.3 + D 58% * 0.5
= 10.6% + 18.6% + 29.0%
= 58.2%

68% - 58.2% = 9.8%p
```

이 숫자를 바로 확정된 효과라고 부르면 안 된다.

하지만 이렇게 말할 수는 있다.

```text
설치 전 흐름을 비슷하게 맞춘 비교 대상과 비교하면,
설치 후 스타 PC방은 약 9%p 정도 더 높아 보인다.
```

## 아무렇게나 섞으면 안 된다

여러 PC방을 섞는다고 해서 아무 숫자나 써도 되는 것은 아니다.

예를 들어 어떤 PC방은 `120%` 넣고, 다른 PC방은 `-20%` 넣는 식으로 계산할 수도 있다.

수학적으로는 이런 계산이 가능할 때가 있다.

하지만 비교 대상으로는 이상하다.

```text
B PC방을 120% 사용한다.
C PC방을 -20% 사용한다.
```

이런 비율은 실제 비교 대상들의 평균으로 보기 어렵다.

그래서 보통은 두 가지 조건을 둔다.

```text
각 비율은 0보다 작으면 안 된다.
모든 비율을 더하면 1이 되어야 한다.
```

그러면 각 PC방을 조금씩 나눠 담은 평균처럼 볼 수 있다.

이 조건은 계산을 조금 답답하게 만든다.

하지만 비교가 너무 이상해지는 것을 막아 준다.

## 컴퓨터는 비율을 어떻게 고를까

비율을 사람이 눈으로 고를 수도 있다.

하지만 PC방이 많아지면 어렵다.

그래서 컴퓨터에게 이렇게 시킨다.

```text
설치 전 스타 PC방과 합성 통제집단의 차이가
가장 작아지는 비율을 찾아라.

단, 각 비율은 0 이상이어야 하고,
비율의 합은 1이어야 한다.
```

작은 예시에서는 직접 확인할 수 있다.

아래 코드는 B, C, D의 비율을 0.1 단위로 바꿔 보면서, 설치 전 차이가 가장 작은 조합을 찾는다.

```python
star_before = [52, 54, 55, 57]

controls_before = {
    "B": [48, 49, 50, 51],
    "C": [56, 57, 59, 60],
    "D": [51, 53, 54, 56],
}

controls_after = {
    "B": [52, 53],
    "C": [61, 62],
    "D": [57, 58],
}

def mix(values_by_pc, weights):
    mixed = []
    weeks = len(next(iter(values_by_pc.values())))
    for week in range(weeks):
        value = 0
        for pc, weight in weights.items():
            value += values_by_pc[pc][week] * weight
        mixed.append(value)
    return mixed

def mean_squared_error(left, right):
    return sum((a - b) ** 2 for a, b in zip(left, right)) / len(left)

best = None

for b in range(11):
    for c in range(11 - b):
        d = 10 - b - c
        weights = {"B": b / 10, "C": c / 10, "D": d / 10}
        synthetic_before = mix(controls_before, weights)
        error = mean_squared_error(star_before, synthetic_before)
        if best is None or error < best[0]:
            best = (error, weights, synthetic_before)

error, weights, synthetic_before = best
synthetic_after = mix(controls_after, weights)
star_after = [66, 68]
effect_after = [star - synth for star, synth in zip(star_after, synthetic_after)]

print(weights)
print([round(x, 1) for x in synthetic_before])
print([round(x, 1) for x in synthetic_after])
print([round(x, 1) for x in effect_after])
```

출력은 대략 이렇게 읽으면 된다.

```text
비율: B 0%, C 20%, D 80%
설치 전 합성 비교 대상: 52.0%, 53.8%, 55.0%, 56.8%
설치 후 합성 비교 대상: 57.8%, 58.8%
설치 후 차이: 8.2%p, 9.2%p
```

코드는 새 개념을 설명하는 것이 아니다.

앞에서 말한 일을 기계적으로 해 본 것이다.

눈으로 고른 비율과 컴퓨터가 고른 비율이 꼭 같을 필요는 없다.

중요한 것은 설치 전 차이가 작아지도록 비율을 정하고, 그 비율을 설치 후에도 그대로 쓴다는 점이다.

## 처치 전에도 안 맞으면 믿기 어렵다

합성 통제집단은 설치 전 흐름이 잘 맞을수록 설득력이 생긴다.

반대로 설치 전부터 잘 맞지 않으면 위험하다.

예를 들어 스타 PC방이 설치 전에는 70%, 72%, 75%, 77%였다고 하자.

그런데 다른 PC방들을 아무리 섞어도 50%대 흐름밖에 못 만든다면 어떻게 될까?

설치 후 차이가 커 보여도 믿기 어렵다.

그 차이는 새 마우스 때문이 아니라, 애초에 비교 대상이 맞지 않아서 생겼을 수 있다.

그래서 합성 통제집단을 볼 때는 설치 후 차이보다 먼저 이것을 본다.

```text
설치 전 흐름이 충분히 비슷한가?
```

이 질문에 답하지 못하면 설치 후 차이도 약해진다.

## 다른 곳에도 가짜로 적용해 본다

한 가지 의심이 남는다.

```text
스타 PC방에서 보인 9%p 차이가 정말 큰 차이일까?
```

자료가 적으면 표준오차 같은 숫자를 평소처럼 계산하기 어렵다.

그래서 다른 방식으로 확인한다.

새 마우스를 설치하지 않은 PC방에도 가짜로 같은 절차를 적용해 보는 것이다.

예를 들어 B PC방을 잠깐 처치받은 곳이라고 가정한다.

그리고 C, D, 스타 PC방의 설치 전 자료를 섞어 B PC방과 비슷한 비교 대상을 만든다.

물론 B PC방에는 실제로 새 마우스가 들어가지 않았다.

그런데도 설치 후에 큰 차이가 자주 나온다면, 스타 PC방의 9%p도 특별하지 않을 수 있다.

반대로 다른 PC방들에서는 작은 차이만 나오고, 스타 PC방에서만 큰 차이가 나온다면 더 설득력이 생긴다.

이런 확인을 **플라시보 검사**라고 부를 수 있다.

여기서 플라시보는 실제 처치가 없던 곳에 처치가 있었다고 가정해 보는 확인이다.

## 이 방법이 말할 수 있는 것

합성 통제집단은 무작위 실험이 아니다.

그래서 모든 의심을 없애지는 못한다.

하지만 맞는 비교 대상 하나가 없을 때 쓸 수 있는 강한 방법이다.

이 방법은 특히 이런 상황에서 도움이 된다.

```text
처치받은 대상이 하나이거나 적다.
자료가 여러 시점에 걸쳐 있다.
처치받지 않은 비교 후보가 여러 개 있다.
처치 전 흐름을 비교할 수 있다.
```

예시는 PC방이었지만, 실제로는 도시, 학교, 지역, 회사 같은 단위에도 많이 쓴다.

중요한 것은 이름이 아니다.

질문은 이것이다.

```text
처치받은 대상이 처치를 받지 않았다면,
어떤 흐름을 보였을 것이라고 비교할 수 있는가?
```

합성 통제집단은 그 흐름을 여러 비교 대상의 가중 평균으로 만든다.

## 한 줄 요약

비교 대상 하나가 딱 맞지 않을 때는 여러 비교 대상을 0 이상이고 합이 1인 비율로 섞어, 처치 전 흐름이 비슷한 가상의 비교 대상을 만든 뒤 처치 후 차이를 본다.
